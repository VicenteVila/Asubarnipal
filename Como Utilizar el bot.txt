📚 ¿A qué información tiene acceso el bot?

    Sistema	                    Qué contiene	                                     Acceso

SQLite (FTS5)	Toda la info indexada — entidades, contenido, tags, fechas	✅ /query busca aquí directamente

FAISS (vector)	Embeddings semánticos de documentos	                        ✅ /query_vectorial busca por significado

Wiki files	Notas .md con frontmatter	                                ✅ Leídas por dashboard y sync Obsidian

Raw sources	Contenido original tal cual se ingirió	                        ✅ Solo lectura, nunca se modifica

H-Mem	        Conversaciones y entidades extraídas	                        ✅ Automático en cada chat


El Librarian busca en SQLite FTS5 (líneas 391-409 de wiki.py). Cuanto más ingieres, más contexto tiene.

❓ ¿Cuántas preguntas puedes hacer?

Ilimitadas, pero con restricciones:
- Contexto LLM: ~32K tokens máximo por respuesta
- Memoria H-Mem: Historial conversacional guardado automáticamente
- Rate limits: Depende de Ollama local o API externa

🗣️ ¿Cómo hacer las preguntas?

         Tipo	                       Ejemplo	                          Mejor comando
Concepto técnico	/query En qué consiste LoRA?	        /query → da respuesta + propuesta
Búsqueda semántica	/query_vectorial modelos de atención	Busca por significado, no palabras
Exploración amplia	/query qué tendencias en IA	        /query → modo exploratorio
Consulta rápida	        Solo escribe sin comando	        Respuesta con contexto H-Mem

🕸️ ¿Para qué son los grafos?

┌──────────────────────────────────────────┐
│ 3 TIPOS DE GRAFO EN EL SISTEMA                         │
├──────────────────────────────────────────┤
│ 1. GRAFO DE CONOCIMIENTO (GraphBuilder)                │
│    Nodos = entidades (personas, papers, conceptos)     │
│    Aristas = wikilinks [[Note1]] → [[Note2]]           │
│    Uso: /hubs (mostrar conceptos más conectados)       │
│         /clusters (comunidades temáticas)              │
│         Dashboard → Grafo tab                          │
├──────────────────────────────────────────┤
│ 2. GRAFO VECTORIAL (RAG/FAISS)                         │
│    Nodos = documentos embebidos                        │
│    Aristas = similarity score entre vectores           │
│    Uso: /query_vectorial (búsqueda semántica)          │
│         Búsqueda por significado, no por palabras      │
├──────────────────────────────────────────┤
│ 3. GRAFO H-Mem (EntityGraph en hybrid_retriever)       │
│    Nodos = entidades extraídas de conversaciones       │
│    Aristas = related_to, works_on, part_of...          │
│    Uso: /entidades (mostrar hubs de entidades)         │
│         Razonamiento multi-hop en conversaciones       │
└──────────────────────────────────────────┘

En resumen:

- Grafo de conocimiento → Te ayuda a ver qué conceptos son centrales y cómo se relacionan las ideas entre sí
- Grafo vectorial → Permite buscar por significado, no por palabras exactas
- Grafo H-Mem → Da al bot memoria de contexto conversacional
